# AGENTS Guide

## Persistent User Preferences
- Use British English in comments, documentation, and assistant outputs.
- Follow PEP 8.
- Do not code defensively.
- Use the project's `.venv` for Python execution and Python tooling.
- Keep notebook comments professional and impersonal; do not include comments directed at the user.
- Do not add AI-related comments in code or notebook cells.
- Keep comments strictly about code behaviour and implementation.
- Do not use first- or second-person pronouns in comments (for example, "we", "us", "you").
- Do not mark instructions or implementation notes as optional; either implement them or omit them.
- Do not use terms such as "optional" in comments.
- Do not add backward-compatibility shims, fallback aliases, or legacy variable bridges unless explicitly requested.
